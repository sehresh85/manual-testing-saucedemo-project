# Manual Testing Project – SauceDemo

This repository contains a manual testing project performed on the SauceDemo e-commerce web application.

## Project Objective
The goal of this project is to demonstrate the manual QA testing process including test case design, execution, defect reporting, and test summary documentation.

## Application Tested
SauceDemo E-commerce Demo Application

## Test Scope
The following modules were tested:

- Login functionality
- Product listing page
- Product sorting
- Cart operations
- Checkout process
- Navigation menu
- Logout functionality

## Test Artifacts

The following testing documents are included in this repository:

- Test Cases (40 manual test cases)
- Test Execution Results
- Bug Reports
- Test Summary Report

## Test Execution Summary

| Metric | Result |
|------|------|
Total Test Cases | 40 |
Passed | 40 |
Failed | 0 |
Exploratory Issues Documented | 6 |
Functional Bugs Identified | 1 |

## Key Finding

During exploratory testing, a functional issue was identified where the system allows order completion even when the cart is empty.
